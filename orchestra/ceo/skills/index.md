# Skill Index

## BUILD ‚Äî New features, components, creative work
- `brainstorming` ‚Äî Explore intent, propose approaches, get design approval. MANDATORY before implementation.
- `writing-plans` ‚Äî Create bite-sized implementation steps with exact file paths and failing tests.
- `test-driven-development` ‚Äî Write failing test first, then minimal code to pass. Non-negotiable.
- `subagent-driven-development` ‚Äî Fresh subagent per task with code review between tasks.
- `frontend-design` ‚Äî Production-grade frontend with high design quality.

## FIX ‚Äî Bugs, test failures, unexpected behaviour
- `systematic-debugging` ‚Äî 4-phase root cause analysis: investigate, pattern, hypothesis, fix.
- `test-driven-development` ‚Äî Write regression test for the bug, then fix.
- `dispatching-parallel-agents` ‚Äî Multiple independent failures? Deploy parallel agents.

## SHIP ‚Äî Deployment, review, branch completion
- `verification-before-completion` ‚Äî Run commands, read output, confirm success BEFORE claiming done.
- `requesting-code-review` ‚Äî Dispatch code-reviewer subagent before merge.
- `receiving-code-review` ‚Äî Verify feedback is correct before implementing.
- `finishing-a-development-branch` ‚Äî Structured options: merge, PR, or cleanup.
- `using-git-worktrees` ‚Äî Isolated workspace for feature branches.

## DESIGN ‚Äî Frontend, UI/UX, visual polish
- `frontend-design` ‚Äî Build distinctive interfaces, avoid generic AI aesthetics.
- `critique` ‚Äî Evaluate visual hierarchy, information architecture, emotional resonance.
- `polish` ‚Äî Final quality pass: alignment, spacing, consistency, detail.
- `audit` ‚Äî Comprehensive audit: accessibility, performance, theming, responsive.
- `simplify` ‚Äî Strip to essence, remove unnecessary complexity.
- `bolder` ‚Äî Amplify safe/boring designs for visual impact.
- `quieter` ‚Äî Tone down aggressive/loud designs.
- `colorize` ‚Äî Add strategic colour to monochromatic interfaces.
- `animate` ‚Äî Purposeful micro-interactions and motion.
- `adapt` ‚Äî Cross-device, cross-platform consistency.
- `harden` ‚Äî Error handling, i18n, edge cases. Production-ready.
- `optimize` ‚Äî Loading speed, rendering, bundle size.
- `onboard` ‚Äî First-time user experience, empty states.
- `clarify` ‚Äî Improve unclear UX copy, error messages, labels, instructions.
- `delight` ‚Äî Add personality, joy, memorable touches to functional interfaces.
- `extract` ‚Äî Consolidate reusable components and patterns into design system.
- `normalize` ‚Äî Match design system, ensure consistency across pages.

## OPERATE ‚Äî Monitoring, stats, scheduling, maintenance
- `loop` ‚Äî Run any prompt/command on a recurring interval.
- `schedule` ‚Äî Cron-scheduled remote agents.
- `claude-api` ‚Äî Build with Anthropic SDK / Claude API.
- `update-config` ‚Äî Modify settings.json for persistent automation.
