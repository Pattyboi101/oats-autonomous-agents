# Skill Chains

Common multi-step workflows. Execute sequentially, loading each skill JIT.
Mask previous skill's output when entering the next phase.

## Build a New Feature
1. `brainstorming` ‚Äî design approval
2. `writing-plans` ‚Äî implementation steps
3. `test-driven-development` ‚Äî red-green-refactor per step
4. `verification-before-completion` ‚Äî confirm everything passes
5. `requesting-code-review` ‚Äî peer review before merge
6. `finishing-a-development-branch` ‚Äî merge/PR/cleanup

## Fix a Bug
1. `systematic-debugging` ‚Äî root cause analysis
2. `test-driven-development` ‚Äî regression test + fix
3. `verification-before-completion` ‚Äî confirm fix holds

## Ship / Deploy
1. `verification-before-completion` ‚Äî all tests pass, no regressions
2. `finishing-a-development-branch` ‚Äî merge or PR

## Polish a Page
1. `critique` ‚Äî identify what needs improvement
2. `frontend-design` ‚Äî implement improvements
3. `polish` ‚Äî final detail pass

## Major Refactor
1. `brainstorming` ‚Äî scope and approach
2. `using-git-worktrees` ‚Äî isolated workspace
3. `writing-plans` ‚Äî step-by-step plan
4. `test-driven-development` ‚Äî refactor with test safety net
5. `requesting-code-review` ‚Äî review before merge
