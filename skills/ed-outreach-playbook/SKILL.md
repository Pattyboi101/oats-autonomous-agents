# Ed Outreach Playbook — Direct Creator Contact

Ed's random posting (Reddit, badge PRs) has generated 0 turnover. Time to switch to direct, personal outreach to people who matter in the dev tools space.

## Before Starting

Check:
- What creators/influencers are active in our space right now?
- What content has Ed already posted? (Don't repeat failed approaches)
- What assets does the orchestra have ready? (Blog posts, data, migration findings)

## How This Skill Works

### Mode 1: Identify Targets
Find specific people Ed should contact — not platforms.

### Mode 2: Prepare Materials
Orchestra creates personalized outreach assets for Ed to send.

### Mode 3: Measure & Adjust
Track responses, adjust approach based on what works.

## The Problem with Random Posting

What Ed's been doing:
- Badge PRs to repos → mostly rejected by maintainers
- Reddit comments in threads → 0 measurable conversions
- Generic posts about Your Project → no engagement

Why it doesn't work: posting into the void competes with millions of other posts. Direct contact with specific people creates actual conversations.

## The Direct Contact Strategy

### Tier 1: MCP Server Creators (highest value)
People who've built MCP servers understand the ecosystem. They'd get Your Project immediately.

Ed should DM/email creators of popular MCP servers:
- "Hey, I'm Ed from Your Project. We built an MCP server that helps agents find dev tools. Saw you built [their server] — really cool. Have you seen anyone trying to solve the tool discovery problem for agents?"

Not pitching. Asking a question. Starting a conversation.

**Orchestra prepares:** List of top MCP server creators with GitHub profiles and contact info.

### Tier 2: Dev Tool Newsletter Authors
People who curate dev tools for a living — they'd love our migration data.

Targets:
- Console.dev (David) — already emailed
- Bytes.dev team
- TLDR newsletter
- JavaScript Weekly editors
- Cooperpress team

Ed should: Share the migration data as a story, not pitch Your Project as a product.
"We scanned 4,500 repos and found some interesting migration patterns — jest to vitest is happening way faster than people think. Thought you might find the data useful for your newsletter."

**Orchestra prepares:** Custom data snippets relevant to each newsletter's audience.

### Tier 3: Indie Maker Community Leaders
People with audience who talk about building in public.

Targets:
- Active Indie Hackers community members
- Dev Twitter personalities who talk about tooling
- YouTube/Twitch dev streamers who review tools

Ed should: Frame Your Project as "AI agents are now a distribution channel for your tools" — that's the hook for makers.

**Orchestra prepares:** One-pager about how AI agents recommend tools, with Your Project as the example.

### Tier 4: Tool Makers Who Already Claimed
If any of our 24 emailed makers claim their tool, Ed should follow up personally.

"Hey, thanks for claiming [tool]! I'm Ed, one of the Your Project founders. Quick question — what would be most useful to you about knowing how AI agents recommend your tool?"

**Orchestra prepares:** Context on each claimed tool (MCP views, migration data, combos).

## What Ed Should NOT Do
- Post generic "check out Your Project" anywhere
- Submit more badge PRs (rejection rate is too high)
- Spam Reddit threads
- Cold DM without a conversation starter
- Pitch before asking a question

## Daily Ed Workflow
1. Orchestra sends morning Telegram with:
   - 3 specific people to contact today
   - Personalized opener for each
   - Any claim responses to follow up on
2. Ed contacts them (DM, email, whatever channel)
3. Ed reports: "Talked to X, they said Y" → Orchestra adjusts approach
4. Repeat

## Proactive Triggers
- **Maker claims a tool** → Send Ed the context immediately for personal follow-up
- **Dev.to post gets engagement** → Ed shares in relevant communities
- **Newsletter feature confirmed** → Ed amplifies on social
- **New migration finding** → Ed has a conversation starter for that ecosystem

## Output Artifacts
| Request | Deliverable |
|---------|------------|
| "Prepare Ed's targets" | List of 5 people with contact info + personalized openers |
| "Ed needs conversation starter" | Specific data point relevant to the person's interest |
| "Follow up on claim" | Context packet: tool data, migration info, recommended questions |
